# magiq-auth Refactor: Comprehensive Implementation Plan

> **For Hermes:** Use subagent-driven-development to execute phases sequentially with two-stage review.

**Goal:** Refactor magiq-auth with modernization (.NET 8), infrastructure improvements (async, caching, redis), security fixes, and dependency cleanup.

**Architecture:** Phase-based execution with stabilization between phases. Each phase contains logically related changes. PRs group tasks within a phase to keep review scope manageable.

**Tech Stack:** .NET 8, Entity Framework Core, Redis (updated), HttpClient, async/await patterns

---

## Branching Strategy

### Main Branch Flow
```
main (production)
  ↓
develop (integration branch)
  ↓
feature/phase-N-* (phase branches)
  ↓
task/* (task branches)
```

### Branch Naming
- **Phase branch:** `feature/phase-1-foundations`
- **Task branch:** `task/phase-1-code-cleanup` (off develop)
- **Hotfix:** `hotfix/security-issue-name` (off main)

### PR Strategy
- **One PR per task or logical group** (keep reviews < 400 lines when possible)
- **All task branches merge to phase branch**
- **Phase branch tested/verified, then merges to develop**
- **develop → main only after full QA**
- **Require 1+ approval before merge**
- **Squash commits on merge** to keep main history clean

}

### Task 4.1: Convert Sync to Async (Part 1: DAL/Repository Layer)
**Branch:** `task/phase-4-async-dal`
**PR Title:** `refactor: convert data access layer to async`
**Objective:** Async data access—prevent thread pool starvation.

**Changes:**
- Identify all sync database calls (`.ToList()`, `.FirstOrDefault()`, etc.)
- Convert to async (`async Task<>`, `await`, `.ToListAsync()`, `.FirstOrDefaultAsync()`)
- Update all callers
- Add cancellation token support

**Pattern:**
```csharp
// Before
public class UserRepository
{
    public User GetById(int id) => _dbContext.Users.FirstOrDefault(u => u.Id == id);
}

// After
public class UserRepository
{
    public async Task<User> GetByIdAsync(int id, CancellationToken ct = default)
        => await _dbContext.Users.FirstOrDefaultAsync(u => u.Id == id, ct);
}
```

**Verification:**
- All DB queries async
- Callers updated
- Integration tests pass
- No `Task.Result` or `.Wait()` calls

### Task 4.4: Fix Thread Blocks and Race Conditions
**Branch:** `task/phase-4-threading-fixes`
**PR Title:** `fix: eliminate thread blocks and race conditions`
**Objective:** Safe concurrent access.

**Changes:**
- Identify lock statements and replace with `ReaderWriterLockSlim` or `SemaphoreSlim`
- Fix races in cache access (use lock or atomic operations)
- Fix races in configuration/initialization
- Add concurrency tests

**Pattern:**
```csharp
// Before (dangerous)
private object _lockObj = new();
private Dictionary<string, string> _cache = new();

public string Get(string key)
{
    lock (_lockObj)
        return _cache[key];
}

// After (better)
private readonly SemaphoreSlim _semaphore = new(1);
private readonly Dictionary<string, string> _cache = new();

public async Task<string> GetAsync(string key)
{
    await _semaphore.WaitAsync();
    try
        return _cache[key];
    finally
        _semaphore.Release();
}
```

**Verification:**
- Lock contention tests
- Race condition stress tests (concurrent access)
- ThreadSanitizer/code review
