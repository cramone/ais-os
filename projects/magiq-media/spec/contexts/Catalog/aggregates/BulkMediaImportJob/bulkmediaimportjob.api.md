# BulkMediaImportJob — API

_Context: `Catalog`_
_Aggregate: `BulkMediaImportJob`_

---

## API Conventions

Cross-cutting concerns follow [`spec/shared/api-conventions.md`](../../../../shared/api-conventions.md).

- **Authentication:** `Authorization: Bearer <jwt>` required on all endpoints.
- **Idempotency:** All mutating endpoints accept `IdempotencyKey: <uuid>`.
- **Errors:** All error responses use `Content-Type: application/problem+json` (RFC 9457).


## Write Endpoints

### `POST /v1/catalog/collections/{collectionId}/media-items/import`

Initiates async bulk media import job. Accepts JSON manifest or CSV. Returns immediately with `jobId`. Worker issues pre-signed upload URLs. Client retrieves URLs via `GET /v1/catalog/import-jobs/{jobId}/upload-urls`, uploads assets, then confirms via `POST /v1/catalog/import-jobs/{jobId}/confirm-uploads`.

**Content-Type:** `application/json | text/csv`

---

#### Format 1: JSON Manifest (`application/json`)

**Request:**
```json
POST /v1/catalog/collections/{collectionId}/media-items/import
Content-Type: application/json

{
  "items": [
    {
      "title": "Hero Image 001",
      "fileName": "hero-001.jpg",
      "folderId": "018f...",
      "recordTypeId": "018g..."
    },
    {
      "title": "Hero Image 002",
      "fileName": "hero-002.jpg",
      "folderPath": "Season 1/Episode 01",
      "recordTypeName": "Marketing Image"
    }
  ]
}
```

**Field Resolution:**
- `folderId` OR `folderPath` required (not both). If `folderPath`: resolved via hierarchy lookup.
- `recordTypeId` OR `recordTypeName` required. If `recordTypeName`: resolved via RecordType lookup.
- `fileName` required — used to generate pre-signed S3 upload URL with correct `Content-Type` and file extension.

---

#### Format 2: CSV (`text/csv`)

**Request:**
```
POST /v1/catalog/collections/{collectionId}/media-items/import
Content-Type: text/csv

title,fileName,folderPath,recordTypeName
Hero Image 001,hero-001.jpg,Season 1/Episode 01,Marketing Image
Hero Image 002,hero-002.jpg,Season 1/Episode 02,Marketing Image
```

**Headers:** First row = column names. Required columns: `title`, `fileName`, `folderPath` OR `folderId`, `recordTypeName` OR `recordTypeId`.

---

**Response `202 Accepted`:**
```json
{
  "jobId": "018f1a2b-3c4d-7e8f-9a0b-1c2d3e4f5a6b",
  "status": "Queued",
  "estimatedItems": 100
}
```

**Errors:**

| Status | Condition |
|---|---|
| `400` | Invalid input format, empty input, missing required fields, or exceeds 50 items (phase 1 limit) |
| `401` | Unauthenticated |
| `403` | Caller does not own collection |
| `404` | `collectionId` not found |
| `413` | Manifest exceeds 10MB |

_Accepts `IdempotencyKey` header._

---

### `GET /v1/catalog/import-jobs/{jobId}/upload-urls`

Returns pre-signed S3 upload URLs for asset uploads. Available after job created and worker issues URLs (typically <5 seconds). Paginated to support large batches.

**Query Parameters:**

| Param | Type | Default | Notes |
|---|---|---|---|
| `pageToken` | `string?` | null | Continuation token |
| `pageSize` | `int?` | 50 | Max 200 |

**Response `200 OK`:**
```json
{
  "items": [
    {
      "index": 0,
      "assetId": "018f...",
      "uploadUrl": "https://media-source.s3.amazonaws.com/...",
      "confirmationToken": "a1b2c3d4...",
      "fileName": "hero-001.jpg",
      "expiresAt": "2026-06-05T07:00:00Z"
    },
    {
      "index": 1,
      "assetId": "018g...",
      "uploadUrl": "https://media-source.s3.amazonaws.com/...",
      "confirmationToken": "e5f6g7h8...",
      "fileName": "hero-002.jpg",
      "expiresAt": "2026-06-05T07:00:00Z"
    }
  ],
  "nextPageToken": "eyJ..."
}
```

**Client Workflow:**
1. Call this endpoint to retrieve batch of URLs
2. Upload each file via `PUT {uploadUrl}` with file binary as body
3. After all uploads: call `POST /v1/catalog/import-jobs/{jobId}/confirm-uploads`

**Errors:** `404` (job not found), `425` (Too Early — URLs not yet generated by worker)

---

### `POST /v1/catalog/import-jobs/{jobId}/confirm-uploads`

Confirms client-side asset uploads complete. Accepts array of confirmation tokens. Advances job to validation phase.

**Request:**
```json
{
  "confirmations": [
    { "index": 0, "confirmationToken": "a1b2c3d4..." },
    { "index": 1, "confirmationToken": "e5f6g7h8..." }
  ]
}
```

**Response `204 No Content`**

**Errors:**

| Status | Condition |
|---|---|
| `400` | Invalid confirmation token or index |
| `404` | Job not found |
| `409` | Job not in `AwaitingUploads` status |
| `422` | Confirmation token does not match generated token for index |

_Accepts `IdempotencyKey` header._

**State Transition:**
- After all confirmations received (tracked by `UploadedCount` = `TotalItems`): worker dispatches `StartBulkMediaImportJobValidationCommand`
- Job status → `Validating`
- Virus scans triggered for all uploaded assets

## Read Endpoints

### `GET /v1/catalog/import-jobs/{jobId}`

Returns job status with phase-level progress.

**Response `200 OK`:**
```json
{
  "jobId": "018f1a2b-3c4d-7e8f-9a0b-1c2d3e4f5a6b",
  "jobType": "MediaImport",
  "collectionId": "018e4c7b-...",
  "status": "Processing",
  "inputFormat": "JSON",
  "phases": {
    "upload": {
      "total": 100,
      "completed": 100
    },
    "validation": {
      "total": 100,
      "completed": 100,
      "failed": 2
    },
    "cataloging": {
      "total": 98,
      "completed": 98
    },
    "processing": {
      "total": 98,
      "completed": 75,
      "failed": 3
    }
  },
  "totalItems": 100,
  "succeededCount": 72,
  "failedCount": 5,
  "createdAt": "2026-06-04T07:00:00Z",
  "completedAt": null
}
```

**Status values:** `Queued | AwaitingUploads | Validating | Cataloging | Processing | Completed | Failed | Cancelled`

**Errors:** `404` (job not found or not owned by caller's tenant)

### `GET /v1/catalog/import-jobs/{jobId}/items`

Returns per-item results with phase-level status. Paginated.

**Query Parameters:**

| Param | Type | Default | Notes |
|---|---|---|---|
| `status` | `string?` | all | Filter: `pending | uploaded | validated | cataloged | processed | failed` |
| `phase` | `string?` | all | Filter: `upload | validation | cataloging | processing` |
| `pageToken` | `string?` | null | Continuation token |
| `pageSize` | `int?` | 100 | Max 500 |

**Response `200 OK`:**
```json
{
  "items": [
    {
      "index": 0,
      "title": "Hero Image 001",
      "fileName": "hero-001.jpg",
      "assetId": "018f...",
      "mediaItemId": "018g...",
      "status": "processed",
      "phase": "processing"
    },
    {
      "index": 1,
      "title": "Hero Image 002",
      "fileName": "hero-002.jpg",
      "assetId": "018h...",
      "status": "failed",
      "phase": "validation",
      "errorCode": "VirusDetected",
      "errorMessage": "Virus detected during scan."
    }
  ],
  "nextPageToken": "eyJ..."
}
```

**Errors:** `404` (job not found)

---

### `DELETE /v1/catalog/import-jobs/{jobId}`

Cancels job. Only allowed if job not yet in processing phase. Items already cataloged remain; in-flight processing stopped.

**Response `204 No Content`**

**Errors:** `404` (job not found), `409` (job in terminal state or past cancellation window)

_Accepts `IdempotencyKey` header._
