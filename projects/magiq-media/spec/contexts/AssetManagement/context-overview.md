# AssetManagement — Context Overview

_Bounded context: `AssetManagement`_
_Service: `AssetManagement` module_

---

## Purpose

Owns the full lifecycle of a single uploaded file — from pre-signed URL issuance through virus scan, processing pipeline, rendition generation, and final storage. The Asset is the raw material of the platform; all downstream consumption (cataloguing, signing, media-registration) depends on Assets being in `Active` status.

---

## Responsibilities

- Issuing pre-signed S3 PUT URLs for client-direct upload
- Recording upload initiation and confirmation
- Coordinating validation (virus scan, format check)
- Driving the async processing pipeline (rendition generation, metadata extraction)
- Managing asset lifecycle: Active, Archived, Deleted
- Enforcing quota checks via Billing ACL before URL issuance
- Publishing integration events for completed/failed processing

---

## Aggregate List

| Aggregate | Description |
|---|---|
| `Asset` | Single uploaded file and all its derived representations. Owns the full ingestion and processing lifecycle. |

---

## Service Boundaries

**Owns:**
- `Asset` aggregate event stream (`asset_*` prefixes in `media-events`)
- `media-assets` and `media-asset-detail` DynamoDB read model tables
- S3 objects in `media-source`, `media-renditions`, `media-documents` buckets

**Does not own:**
- MediaItem cataloguing logic (→ Catalog)
- Processing pipeline execution (→ Processing)
- Rendition generation infrastructure (→ Processing)

**Rules:**
- Pre-signed URLs are issued by the Ingest API (in-process command dispatch via MediatR)
- Processing publishes integration events; this context subscribes and dispatches its own commands — no cross-BC command dispatch
- `AssetId` is generated by the caller (UUID v7) for idempotent upload initiation

---

## External Dependencies

| Dependency | Direction | Pattern |
|---|---|---|
| Identity | Upstream | JWT → `OwnerId` via `IdentityAcl`. No runtime calls. |
| Billing / Quotas | Upstream | Sync HTTP quota check before pre-signed URL issuance. Items whose `MediaProfile` lacks `Processing` capability are quota-exempt. Resolved from `media-items` read model. |
| Catalog (MediaItem) | Internal cross-context | `MediaItemId` is nullable — an Asset can exist without a `MediaItem`. When set, capabilities are resolved from the MediaItem's media-profile via `media-items` read model. |
| Processing | Internal cross-context | Processing publishes `ProcessingJobScanResultIntegrationEvent` (and job-level lifecycle events) to `media-integration-events` SNS. AssetManagement subscribes via `media-cross-module-events` and dispatches its own commands (`RecordValidationResultCommand`, `StartAssetProcessingCommand`, etc.) to advance the `Asset` aggregate. |
| Notifications | Downstream | Integration events published for processing completion/failure. |
| Billing | Downstream | `media.asset.processing-completed` published only when `Processing` capability is present. |

### Published Integration Events

Published inline by `AssetDomainEventPublisher` (`AssetManagement.WriteModel`) immediately after the corresponding domain event is persisted. All events are published to the `media-integration-events` SNS topic.

| C# Record Type | Trigger Domain Event | Consumers |
|---|---|---|
| `AssetUploadInitiatedIntegrationEvent` | `AssetUploadInitiated` | Processing (capability-gated), Notifications |
| `AssetValidationPassedIntegrationEvent` | `AssetValidationPassed` | SagaOrchestrator (`AssetValidationPassedSagaHandler`) — advances `AssetIngestionSaga`; carries `HasProcessingCapability` flag so saga selects `StartProcessingJobCommand` or `BypassProcessingJobCommand` without a cross-BC lookup |
| `AssetProcessingCompletedIntegrationEvent` | `AssetProcessingCompleted` | Notifications, Billing (filtered: `Processing` capability only) |
| `AssetProcessingFailedIntegrationEvent` | `AssetProcessingFailed` | Notifications, SagaOrchestrator |
| `AssetAttachedToMediaItemIntegrationEvent` | `AssetAttachedToMediaItem` | Search/Discovery |
| `AssetArchivedIntegrationEvent` | `AssetArchived` | Notifications, Billing |
| `AssetDeletedIntegrationEvent` | `AssetDeleted` | Notifications, Search/Discovery |
| `AssetInfectionDetectedIntegrationEvent` | `AssetInfectionDetected` | Notifications; Security audit log |

### Consumed Integration Events

AssetManagement consumes integration events via the `media-cross-module-events` SQS queue. Consumers are registered in the `Media.IntegrationEventConsumers.Lambda` host.

**From Catalog** — consumer: `MediaItemCapabilityIndexConsumer`

| Event | Handling |
|---|---|
| `MediaItemCreatedMessage` | Materialises a `media-item-capability-refs` projection entry so AssetManagement can resolve capabilities and `MaxFileSizeBytes` without querying Catalog |
| `MediaItemArchivedMessage` | Flips `IsArchived = true` on the projection entry; prevents new uploads against archived media-items |

**From Processing** — consumers: `ProcessingJobScanResultEventHandler`, `ProcessingJobStartedEventHandler`, `ProcessingJobCompletedEventHandler`, `ProcessingJobFailedEventHandler`, `ProcessingJobBypassedEventHandler`

Dispatches the corresponding AssetManagement command to advance the `Asset` aggregate state machine. See [Processing context](../Processing/context-overview.md) for event contracts.

| Event | Handler | Command Dispatched | Asset Transition |
|---|---|---|---|
| `ProcessingJobScanResultIntegrationEvent` | `ProcessingJobScanResultEventHandler` | `RecordValidationResultCommand` — handler resolves `HasProcessingCapability` via `IMediaItemCapabilityService` | Emits `AssetValidationPassed` (or failure variant) |
| `ProcessingJobStartedIntegrationEvent` | `ProcessingJobStartedEventHandler` | `StartAssetProcessingCommand` | `Validating → Processing` |
| `ProcessingJobCompletedIntegrationEvent` | `ProcessingJobCompletedEventHandler` | `CompleteAssetProcessingCommand` | `Processing → Active`; renditions + metadata attached |
| `ProcessingJobFailedIntegrationEvent` | `ProcessingJobFailedEventHandler` | `FailAssetProcessingCommand` | `Processing → ProcessingFailed` |
| `ProcessingJobBypassedIntegrationEvent` | `ProcessingJobBypassedEventHandler` | `ActivateDocumentAssetCommand` | `Validating → Active` (document fast-exit) |

## Ubiquitous Language

| Term | Meaning |
|---|---|
| Asset | A single uploaded file and all its derived representations (renditions, metadata) |
| Rendition | A processed derivative of the original (thumbnail, compressed, transcoded) |
| Ingestion | The full pipeline from upload receipt to `Active` status |
| Shard | Last 4 hex chars of UUID v7 `AssetId` — S3 key partition prefix |
| Validation | Virus scan + format check performed on `ConfirmAssetUpload` |
| Processing | Async pipeline generating renditions and extracting technical metadata |
| Standalone Upload | An Asset uploaded without a `MediaItemId` — full pipeline runs by default |
| Document Asset | An Asset whose owning `MediaItem`'s `MediaProfile` lacks `Processing` capability — fast-exits after virus scan, stored in `media-documents` |
| Pre-signed URL | A time-limited S3 PUT URL issued to the client for direct upload; 15-minute TTL |

## Integration Event Contracts

### Published

#### `AssetUploadInitiatedIntegrationEvent`

**Publisher:** `AssetDomainEventPublisher` — triggered by `AssetUploadInitiated`  
**SNS message type:** `media.asset.upload-initiated`

```csharp
record AssetUploadInitiatedIntegrationEvent(
    string TenantId,
    string AssetId,
    string OwnerId,
    string FileName,
    string ContentType,    // MediaContentType enum value as string
    string StorageKey,
    string Status,         // Always "Pending"
    DateTimeOffset UploadedAt,
    long EventVersion
);
```

---

#### `AssetUploadConfirmedIntegrationEvent`

**Publisher:** `AssetDomainEventPublisher` — triggered by `AssetUploadConfirmed`  
**SNS message type:** `media.asset.upload-confirmed`

```csharp
record AssetUploadConfirmedIntegrationEvent(
    string TenantId,
    string AssetId,
    string OwnerId,
    string FileName,
    string ContentType,    // MediaContentType enum value as string
    string StorageKey,     // S3 key path — used by AssetValidationWorker to locate the object
    string Status,         // Always "Validating"
    DateTimeOffset ConfirmedAt,
    long EventVersion
);
```

> `AssetValidationWorker` (Processing context) subscribes via the `media-processing` SQS queue and uses `StorageKey` and `ContentType` to run the virus scan and format check. `Status` is always `"Validating"` — the asset transitions from `Pending` to `Validating` on this event.

---

#### `AssetValidationPassedIntegrationEvent`

**Publisher:** `AssetDomainEventMapper` — triggered by `AssetValidationPassed`  
**SNS message type:** `media.asset.validation-passed`

```csharp
record AssetValidationPassedIntegrationEvent(
    string TenantId,
    string AssetId,
    string? MediaItemId,          // null for standalone uploads
    string JobId,                 // correlates with the ProcessingJob created on upload
    string StorageKey,            // bucket + key — avoids a secondary lookup by the Processing Worker
    string ContentType,           // MIME type string
    bool HasProcessingCapability, // resolved by RecordValidationResultHandler; carried so the saga needs no cross-BC lookup
    DateTimeOffset PassedAt,
    long EventVersion
);
```

> **Consumer:** SagaOrchestrator (`AssetValidationPassedSagaHandler`) — reads `HasProcessingCapability` directly from the event to advance `AssetIngestionSaga` to either `BypassProcessingJobCommand` (document fast-exit, `false`) or `StartProcessingJobCommand` (full pipeline, `true`). No cross-BC capability lookup is needed in the saga or Processing Worker.

---

#### `AssetProcessingCompletedIntegrationEvent`

**Publisher:** `AssetDomainEventPublisher` — triggered by `AssetProcessingCompleted`

```csharp
record AssetProcessingCompletedIntegrationEvent(
    string TenantId,
    string AssetId,
    string OwnerId,
    IReadOnlyList<string> RenditionStorageKeys,
    DateTimeOffset CompletedAt
);
```

> `RenditionStorageKeys` is the projection of `Rendition.StorageKey` from all renditions attached to the asset. Empty for document media-assets (no `Processing` capability).

---

#### `AssetProcessingFailedIntegrationEvent`

**Publisher:** `AssetDomainEventPublisher` — triggered by `AssetProcessingFailed` and `AssetValidationFailed` (see below).

```csharp
record AssetProcessingFailedIntegrationEvent(
    string TenantId,
    string AssetId,
    string OwnerId,
    string ContentType,        // MediaContentType enum value as string
    string Status,             // AssetStatus enum value as string — see values below
    string FailureCategory,    // FailureCategory enum value as string — see values below
    string Reason,
    DateTimeOffset OccurredAt,
    long EventVersion
);
```

**`Status` values:**

| Value | Source domain event | Meaning |
|---|---|---|
| `"ProcessingFailed"` | `AssetProcessingFailed` | Pipeline failure (timeout, processing error, upload expiry) |
| `"ValidationFailed"` | `AssetValidationFailed` | Virus scan or format check failed |

**`FailureCategory` values:**

| Value | Source domain event | Meaning |
|---|---|---|
| `"ProcessingError"` | `AssetProcessingFailed` | Pipeline worker returned a failure result |
| `"ProcessingTimeout"` | `AssetProcessingFailed` | Processing window expired before completion |
| `"ValidationTimeout"` | `AssetProcessingFailed` | Validation did not complete within the allowed window |
| `"UploadExpired"` | `AssetProcessingFailed` | Upload TTL elapsed; asset never left Pending |
| `"ValidationError"` | `AssetValidationFailed` | Virus scan or format validation rejected the file |

> `AssetValidationFailed` maps to this event (rather than a dedicated event) so the `AssetIngestionSaga` can compensate via its existing `OnAssetProcessingFailed` handler — the saga does not distinguish between validation and pipeline failures for compensation purposes. `AssetInfectionDetected` publishes `AssetInfectionDetectedIntegrationEvent` separately and is not routed through this event.

SagaOrchestrator subscribes to this event to mark the `AssetIngestionSaga` as `Failed` and trigger any necessary compensation.
